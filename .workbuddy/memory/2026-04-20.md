# 2026-04-20 工作记录

## 大文件断点续传功能

- 后端 `src/app.js` 新增分片上传 API：
  - `POST /api/upload/init` — 初始化上传会话，返回 uploadId
  - `GET /api/upload/:uploadId/status` — 查询已上传分片（断点续传）
  - `POST /api/upload/:uploadId/chunk` — 上传单个分片（Busboy 流式解析，直写磁盘，绕过 formidable）
  - `POST /api/upload/:uploadId/merge` — 合并所有分片为完整文件（`stream/promises.pipeline`）
  - 分片临时文件存放在 `src/chunks/<uploadId>/` 目录，合并后自动清理
  - uploadSessions 使用内存 Map 记录会话状态（服务重启后从磁盘恢复）
  - 同步 I/O 全部替换为 `fs.promises`
- 前端新建 `public/upload.html`：
  - 分片大小滑块（10/25/50/100 MB，默认 100 MB）
  - 并发数配置（1/2/3/5/8/10，默认 10）
  - `inFlight` Map 追踪并发分片实时进度，保证进度条准确
  - `speedDisplay` 字段保存有效速度值，避免速度闪烁
  - 支持暂停/继续、断点续传、全部开始/暂停
  - 全局统计栏：文件总数、已完成、总大小、实时速度
- `public/index.html` 导航栏增加「🚀 大文件」链接跳转到 upload.html
- 文件名保留原名（`uploadId_原名.扩展名`）
