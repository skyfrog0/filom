# 2026-04-21 工作记录

## 1. Cluster 多进程架构

- 新增 `src/master.js` 作为 Cluster 主入口
- 主进程监听端口，启动 CPU 核心数 - 1 个 Worker 进程
- Worker 崩溃时自动重启
- `src/app.js` 导出 Koa app 实例供 master.js 使用
- 分片上传请求由 Cluster 自动分发到不同 Worker

## 2. 会话持久化（多进程支持）

- 新增 `src/sessions.json` 持久化上传会话状态
- 每个 Worker 独立内存缓存 `uploadSessions` Map
- 状态变更时自动保存到磁盘
- `/status` API 支持从磁盘扫描已上传分片（Worker 内存丢失时兜底）
- 新增 `DELETE /api/upload/cleanup` 清理 24 小时以上的过期会话

## 3. localStorage 持久化（前端刷新恢复）

- `public/upload.html` 新增 localStorage 持久化机制
- 页面加载时自动恢复未完成任务（`restoreTasks()`）
- 任务状态：`pending` | `uploading` | `paused` | `done` | `error` | `waiting_file`
- 刷新后任务变为 `waiting_file` 状态，等待用户重新选择文件
- 重新选择文件后自动从后端拉取断点状态，继续上传
- 文件匹配依据：文件名 + 文件大小 + uploadId

## 4. Bug 修复

- 修复 koa-body 与 Busboy 冲突问题：对分片上传路由跳过 koaBody 中间件
